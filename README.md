# Avatar Service

Тестовое задание: сервис генерации говорящего аватара по тексту.

## Возможности

* Генерация речи с помощью **Silero TTS**
* Генерация видео с помощью **Wav2Lip**
* Асинхронная обработка через **Celery**
* Очередь задач на **Redis**
* Хранение статусов задач в **SQLite**
* Кэширование результатов по тексту
* REST API на **FastAPI**

---

## Архитектура

```
Client
   │
   ▼
FastAPI
   │
   ▼
Celery + Redis
   │
   ▼
GenerationPipeline
   ├── Silero TTS
   └── Wav2Lip

```

## API

### Создать задачу

```
POST /api/generate
```

Request:

```json
{
  "text": "Привет!"
}
```

Response:

```json
{
  "task_id": "xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx"
}
```

---

### Получить статус

```
GET /api/status/{task_id}
```

Response:

```json
{
  "task_id": "...",
  "status": "COMPLETED"
}
```

Возможные статусы:

* `PENDING`
* `PROCESSING`
* `COMPLETED`
* `FAILED`

---

### Получить результат

```
GET /api/result/{task_id}
```

Возвращает готовый MP4-файл.

---

## Как работает генерация

1. Клиент отправляет текст.
2. FastAPI создает задачу Celery.
3. Celery запускает `GenerationPipeline`.
4. Silero TTS синтезирует речь.
5. Wav2Lip создает видео с синхронизацией губ.
6. Статус задачи обновляется в SQLite.
7. Клиент получает готовый MP4.

---

## Кэширование

Для каждого текста вычисляется SHA-256 хэш.

Если видео для такого текста уже было успешно сгенерировано ранее, сервис возвращает существующий `task_id` без повторного запуска генерации.

---

## Обработка ошибок

При возникновении ошибки:

* задача получает статус `FAILED`;
* текст ошибки сохраняется в базе данных;
* клиент получает соответствующий HTTP-ответ.

---

## Ограничения

* Используется один заранее подготовленный аватар.
* Генерация выполняется на CPU.
* SQLite используется как локальное хранилище статусов задач.

## Почему выбраны именно эти библиотеки

### FastAPI

Использован для реализации REST API благодаря высокой производительности, встроенной поддержке Pydantic и автоматической генерации OpenAPI/Swagger-документации.

### Celery

Используется для выполнения длительных задач в фоновом режиме. Генерация видео занимает несколько секунд, поэтому выполнение через очередь позволяет не блокировать HTTP-запросы.

### Redis

Используется как брокер сообщений и backend для Celery. Простое и надежное решение для организации очередей задач.

### SQLAlchemy + SQLite

SQLAlchemy используется как ORM для хранения информации о задачах и их статусах. В качестве базы данных выбрана SQLite, поскольку она не требует отдельного сервера и полностью достаточна для тестового задания.

### Silero TTS

Выбран для синтеза речи благодаря хорошему качеству русскоязычного TTS, простоте интеграции и возможности локального запуска без использования внешних API.

### Wav2Lip

Используется для синхронизации движений губ с аудиодорожкой. Это одно из наиболее распространенных открытых решений для lip-sync, демонстрирующее хорошее качество генерации.

### Docker

Используется для воспроизводимого запуска сервиса и упрощения развертывания независимо от окружения.
